### Status

Aceito

### Contexto

A plataforma precisa suportar múltiplos produtos SaaS com o máximo de reaproveitamento e baixo custo operacional.

### Decisão

Utilizar um Monólito Modular com DDD (*Domain-Driven Design*), Clean Architecture e Portas e Adaptadores Hexagonais.

### Consequências

- Desenvolvimento mais rápido
- Limites de módulos claros
- Facilidade para uma futura extração para microsserviços
